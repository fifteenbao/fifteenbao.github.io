iPhone + Notion自动化复盘

Dec 24, 2025

在折腾了飞书之后，又回来折腾notion，其实有考虑要不要整合在一个平台上，不过飞书webhhok似乎只能传输字段，再加上自定义的小组件没有那么全面，决定还是在Notion上面做复盘。

操作流程

前期准备

首先需要获取 Notion API , 然后复制数据库的ID： www.notion.so{workspace_name}/{database_id}?v={view_id}

Notion模版

复盘的模版参考了 这个

苹果快捷方式

添加获取URL参考： Notion API文档 Create a page 中 shell 的格式，{} 就是词典、[] 就是数组、“” 就是文本

复盘大纲

至于复盘的内容，用 Gemini 写了一个大纲

第一阶段：客观数据（状态回顾）

  • 日期、领域、评分、是否锻炼、是否读书。
  • 作用：快速勾选，建立当天的基本面。

第二阶段：主观感受（复盘核心）

  • 今天的感受（Feelings）、出现问题（Problem）、继续保持（Keep）。
  • 建议：先写 Problem 再写 Keep。先面对问题，再用亮点收尾，心情会更积极。

第三阶段：行动转化（未来转化）

  • 努力改善（Try）。
  • 作用：这是复盘的唯一输出物，必须是具体的动作。

BTY, 把这个模版上架了，感觉操作起来比飞书难很多，还是值得 ¥1.88，期待有人下单。

在 Hugo 使用 Obsidian 编辑

May 24, 2023

导入

左侧边栏 导入其他库 > 打开库文件 ，导入 hugo文件夹，其实只有 /content/post 用得到，想直接使用hugo自定义的模版就导入所有目录。

头字段 FrontMatter

Markdown 最前面的头字段，类似于这样

title = 'spf13-vim 3.0 release and new website'
date = '2023-05-24' 
categories = ['Development', 'VIM'] 
description = 'spf13-vim is a cross platform distribution of vim plugins and resources for Vim.' 
slug = 'spf13-vim-3-0-release-and-new-website' 
tags = ['.vimrc', 'plugins', 'spf13-vim', 'vim']

通常储模板文件存在 /archetypes 中，用hugo命令 hugo new posts/XXX.md 就会自动搜索和套用， HUgo 官方教程

在 Obsidian 设置 > 核心插件 > 模版 添加 /archetypes ，就可以在左侧边栏直接插入头字段了。
