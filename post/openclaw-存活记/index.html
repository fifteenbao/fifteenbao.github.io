OpenClaw 存活记

Mar 30, 2026 · fifteenbao

从 2 月入坑 OpenClaw 到现在，已经从最初的"尝鲜"变成了"存活"状态。整理一下这段时间收集到的配置、技能和踩过的坑，给想入坑的朋友一个参考。

一、为什么是 OpenClaw

最初被 OpenClaw/MoltBot/ClawdBot 吸引是因为它的低门槛和高扩展性：

  • 一键部署脚本，新手友好
  • 支持多种大模型接入
  • 丰富的 Channel（接入渠道）和 Skill（技能）生态
  • 可以控制硬件（机械臂！）

但真正开始玩之后才发现，Token 消耗和硬件校准才是存活的关键。

二、服务器部署方案对比

云服务器方案

云服务商 优势 问题 推荐指数
阿里云 大模型 API 默认开通，网络稳定 按量付费容易欠费（第一天欠费 8 块…） ⭐⭐⭐
腾讯云 版本更新快，channel/skill封装好 莫名崩溃，稳定性一般 ⭐⭐
自建服务器 成本可控，数据本地化 需要公网 IP/内网穿透 ⭐⭐⭐⭐

当前方案：阿里云轻量应用服务器 + 大模型 API

三、模型选择与 Token 降本增效

模型对比

模型 特点 适用场景 Token 消耗
qwen3 max 响应快，理解能力好 日常对话、简单任务 高（随随便便大几十万）
deepseek r1/v3.2 性价比高 代码生成、逻辑推理 中
GLM4.7 中文理解好 文档处理 中
claude 最强理解力 复杂任务调试 最高

降本增效策略

  1. 分层使用：日常用 qwen3 max（7.9/月 coding-plan），复杂任务切换到 Claude
  2. 调试通后切换：用贵模型调试通流程，再切回便宜模型
  3. 定期清理记忆：切换模型后重启 Gateway，避免记忆混乱
  4. 本地缓存：常用回复模板本地存储，减少 API 调用

💡 强烈建议阅读：Token 降本增效相关文档

校准流程

参考：https://wiki.seeedstudio.com/lerobot_so100m_new/

  1. 按照 Seeed 教程安装本地环境
  2. 机械校准（非常重要！）
  3. 校准文件提供给 OpenClaw 参考执行
  4. 单电机测试 → 多电机联动测试

⚠️ 注意：校准不当会导致电机损坏，我坏了两个电机……

软件部署

# 一键安装脚本
curl -fsSL https://openclaw.ai/install.sh | bash

# 重启 Gateway
openclaw gateway restart

权限配置

2026.3.2 版本后，默认只有 message 权限：

  }
}

八、踩坑记录

1. Token 爆炸

  • 问题：qwen3 max 随随便便大几十万 token
  • 解决：分层使用模型 + 定期清理记忆

2. 电机损坏

  • 问题：校准不当导致两个电机烧毁
  • 解决：严格按照官方校准流程，先用低功率测试

3. 板子连接失败

  • 问题：LED 灯环配置后板子无法连接
  • 可能原因：GPIO 冲突/供电不足
  • 解决：待排查（板子已凉……）

4. 飞书权限

  • 问题：机器人无法创建飞书文档
  • 解决：需要手动在飞书开放平台审批权限
